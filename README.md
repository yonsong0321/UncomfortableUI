# 🎮 UUI 프로젝트 (4인 모바일 멀티플레이 파티 게임)

본 프로젝트는 4인의 플레이어가 각자의 모바일 기기로 접속하여 즐기는 **1 vs 1 vs 1 vs 1 개인전 기반 네트워크 파티 게임**입니다. 

## 📝 게임 기획 및 기본 흐름 (4라운드 로테이션)

우리 게임의 핵심 재미 요소는 **'치열한 개인전 속에 매 라운드 난입하는 1명의 방해꾼'**입니다. 누군가와 협력하는 팀전이 아닌, 철저히 각자도생하는 서바이벌 구조입니다.

* **참여 인원:** 4인 (각자 기기 사용, 1 vs 1 vs 1 vs 1)
* **진행 방식 (총 4 Stage):** * 4개의 스테이지를 진행하는 동안, **매 스테이지마다 1명이 '방해꾼(Saboteur)'**이 되고 나머지 3명은 '일반 플레이어'가 되어 게임을 진행합니다.
    * **일반 플레이어의 경쟁:** 3명의 일반 플레이어는 절대 협력하지 않습니다. 각자 가장 높은 점수(또는 가장 빠른 클리어)를 기록하여 1등을 차지하기 위해 서로 경쟁합니다.
    * **방해꾼의 역할:** 방해꾼은 나머지 3명의 플레이를 훼방 놓아 판을 뒤흔드는 조커 역할을 합니다.
    * **공평한 로테이션:** 서버 접속 순서대로 부여받은 플레이어 번호(P1~P4)를 기준으로, 1라운드는 P1이 방해꾼, 2라운드는 P2가 방해꾼... 식으로 총 4번 순환합니다. 4명 모두가 정확히 한 번씩 방해꾼 역할을 수행하게 됩니다.

## 👥 팀 구성 및 1차 개발 목표 (Milestone)

본격적인 미니게임 콘텐츠(스테이지)를 만들기 전, 안정적인 게임 구동을 위해 팀원 6명이 2개의 팀으로 나뉘어 **'기반 시스템'**을 먼저 구축합니다.

### 🌐 서버(Server) 팀 (3인)
**"4명이 한 방에 모여 순서를 부여받는 로비 시스템 구축"**
* **Firebase 세팅:** 데이터베이스 연동 및 실시간 통신 환경 구축.
* **로비(Lobby) 구현:** 방 생성/참가, 4명 접속 시 'Ready' 상태 동기화.
* **플레이어 식별:** 접속 순서에 따라 Player 1 ~ 4 고유 식별자(ID) 부여. (이 ID가 방해꾼 순서의 기준이 됩니다.)

### 📱 클라이언트(Client) 팀 (3인)
**"서버가 붙기 전, 게임 화면이 매끄럽게 돌아가는 뼈대 구축"**
* **코어 로직 (`MainActivity`):** 스테이지 1~4까지 화면이 차례대로 넘어가며 방해꾼 역할이 순환하는 로테이션 로직 구현.
* **더미(Dummy) 테스트:** 내용이 없는 빈 스테이지 화면들을 연결하여 화면 전환 시 에러가 없는지 테스트.
* **공통 유틸리티:** 전체 UI 테마(색상/폰트) 확립, 효과음/BGM 재생을 위한 `SoundManager` 세팅.

## 📂 프로젝트 폴더 구조

패키지 충돌을 막기 위해 역할을 철저히 분리했습니다. `org.androidtown.uui` 패키지 구조는 다음과 같습니다.

| 패키지명 | 역할 및 설명 | 주요 파일 |
| :--- | :--- | :--- |
| **`core`** | **게임의 심장.** 스테이지 이동, 4라운드 방해꾼 로테이션 처리, Firebase 통신 등 앱 전체 흐름을 제어합니다. | `MainActivity.java`, `NetworkManager.java` |
| **`stages`** | **실제 게임 화면.** 향후 추가될 다양한 미니게임 액티비티들이 모이는 곳입니다. | `Stage01Activity.java`, `Stage02Activity.java` 등 |
| **`utils`** | **공통 기능.** 사운드 관리 등 앱 전체에서 공용으로 쓰는 도구들입니다. | `SoundManager.java` |

## 💻 스테이지(Activity) 개발 가이드 (필독)

추후 클라이언트 팀이 뼈대를 완성하고 나면, 모든 팀원이 이 규칙을 따라 미니게임(스테이지)을 개발하게 됩니다.

### 1. 방해꾼 여부 확인 (Input)
모든 스테이지는 화면이 켜질 때 `Intent`를 확인하여 **현재 내 기기가 이번 라운드의 방해꾼인지 일반 플레이어인지** 판단하고 화면을 분기해야 합니다.
```java
// 스테이지 Activity의 onCreate 내부
boolean isSaboteur = getIntent().getBooleanExtra("IS_SABOTEUR", false);

if (isSaboteur) {
    // TODO: 방해꾼 전용 화면(XML) 세팅 및 방해 로직 실행
} else {
    // TODO: 일반 플레이어용 화면(XML) 세팅 및 클리어 로직 실행 (개인전)
}
```

### 2. 점수/시간 반환 (Output)
스테이지 종료 시, 해당 기기에서 획득한 점수나 소요된 **시간(밀리초)**을 `MainActivity`로 돌려주고 액티비티를 종료해야 합니다.
```java
// 게임 종료 시 호출하는 메서드 내부
long timeTakenMs = 15000; // 예시: 15초 걸림

Intent resultIntent = new Intent();
resultIntent.putExtra("CLEAR_TIME", timeTakenMs);
setResult(RESULT_OK, resultIntent);
finish(); // 스테이지 종료 후 MainActivity(대기실/정산 화면)로 돌아감
```
